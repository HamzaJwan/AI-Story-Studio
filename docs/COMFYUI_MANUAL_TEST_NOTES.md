# ComfyUI Manual Test Notes

Last updated: 2026-06-25

## Purpose

Document Hamza's manual ComfyUI image-generation lessons from the AI Server so the product path can use a curated, repeatable MVP workflow instead of exposing raw ComfyUI template complexity to users.

This file records observations only. It does not commit generated PNGs, model files, workflow outputs, credentials, or server addresses.

## AI Server Context

Hamza opened ComfyUI manually on the AI Server and tested workflows from the built-in/template UI.

Observed result:

- ComfyUI is running and usable on the AI Server.
- `sd_xl_base_1.0.safetensors` is present and works.
- `SDXL Simple` is the easiest practical starting template.
- Some public/template workflows request checkpoints that are not installed locally.

Examples of missing models seen in templates:

- `sd_xl_refiner_1.0.safetensors`
- `sd_xl_turbo_1.0_fp16.safetensors`
- `DreamShaper_8_pruned.safetensors`

Important lesson: a "missing model" warning often means the selected template expects a different checkpoint. It does not mean ComfyUI itself is broken, and it does not mean the project should download every missing model.

## Workflow Used

Recommended MVP starter workflow from the manual test:

- Template: `SDXL Simple`
- Checkpoint: `sd_xl_base_1.0.safetensors`
- Resolution: `768x768` for first stable tests
- Batch size: `1`
- Steps: around `20`
- CFG: around `6.5` to `7`
- Avoid Refiner initially
- Avoid Turbo initially
- Avoid downloading extra checkpoints until a benchmark justifies them
- Avoid running many image jobs while another executor/autopilot is using the same ComfyUI service

## Missing Refiner Issue

One workflow requested:

```text
sd_xl_refiner_1.0.safetensors
```

That checkpoint was not installed.

The manual workaround that succeeded:

1. Find the node named approximately `Load Checkpoint - REFINER`.
2. Change the checkpoint from `sd_xl_refiner_1.0.safetensors` to `sd_xl_base_1.0.safetensors`.
3. Re-run the workflow.

After this change, the missing-model error disappeared and the workflow ran.

This workaround is acceptable for manual experiments, but it is not the ideal product pipeline. For AI Story Studio MVP image generation, prefer a curated workflow that does not require Refiner. Treat Refiner as an optional future benchmark, not a Phase 2.1/2.2 blocker.

## Gender / Character Drift Issue

A follow-up prompt using wording like `same old Libyan storyteller` produced a different character and even changed the person from male to female in one result.

This confirms an important product risk:

- Prompt-only continuity is not enough.
- `same character` wording does not guarantee continuity.
- Gender, age, identity, clothing, and visual anchors must be repeated explicitly in each prompt.
- A negative prompt should actively block gender/identity drift.
- Strong continuity later needs a reference workflow such as IP-Adapter, ControlNet, or another character-anchor method.

## Prompting Lessons

A stronger positive prompt should repeat fixed identity terms:

- elderly Libyan man
- male
- old man
- weathered masculine face
- subtle gray beard
- traditional robe
- wrapped headscarf
- same wooden window
- same warm lantern
- cinematic realistic photo
- warm orange lantern light
- dark blue night
- realistic face
- natural hands

A stronger negative prompt should explicitly block drift and artifacts:

- female
- woman
- elderly woman
- feminine face
- makeup
- lipstick
- young man
- cartoon
- anime
- sketch
- drawing
- engraving
- distorted face
- bad hands
- extra fingers
- watermark
- text
- logo

## Recommended MVP Image Workflow

For product integration, AI Story Studio should not expose all ComfyUI templates directly. Use a backend-controlled curated workflow:

1. Use SDXL Base first.
2. Generate one scene image at a time.
3. Keep batch size at `1`.
4. Store prompt, negative prompt, seed, model, dimensions, and workflow metadata.
5. Serve outputs through the backend only.
6. Keep generated images in ignored data folders.
7. Let Hamza review/regenerate images before treating them as final.

## What Not To Download Now

Do not download every missing checkpoint shown by ComfyUI templates.

Not required for MVP:

- SDXL Refiner
- SDXL Turbo
- DreamShaper
- Additional style checkpoints
- ControlNet/IP-Adapter models before a dedicated continuity benchmark

## Project Implications

- SDXL Base is enough for first MVP image generation.
- Refiner is optional and should not block Phase 2.1/2.2.
- Missing model warnings usually belong to the selected template, not the whole service.
- The product should use curated backend workflows, not raw public ComfyUI templates.
- Character continuity cannot rely on prompt text alone.
- The product needs style presets, character bible, gender lock, location/object bible, negative prompts, seed metadata, and later reference-image workflows.
