# Safety & Rights Checklist

Last updated: 2026-06-25

## Purpose

AI Story Studio will eventually handle voices, reference images, character anchors, generated images, and videos. Every reference asset needs clear rights and consent metadata before it becomes part of a production workflow.

This is not part of Phase 1.5 implementation, but the rule is non-negotiable for future voice/image expansion.

## Voice Checklist

- Is the voice licensed?
- Is it a synthetic voice, public/community model, or a real-person clone?
- If it is a real person, is there explicit consent?
- Is the usage scope documented?
- Is the source/model name documented?
- Is there a restriction against commercial use?
- Is it a celebrity/public-figure voice? If yes, do not use without explicit written permission.

## Image Reference Checklist

- Does the user own the image?
- Does the user have permission to use the person/object/location?
- Is the image of a real person?
- Is the use consent-safe?
- Is it a celebrity/public figure? If yes, do not use without explicit permission.
- Is the asset source documented?

## Model / Engine Checklist

- Benchmark verdict recorded?
- License known?
- Output sample saved outside Git?
- Known limitations documented?
- Hardware tested on the AI Server?

## Project Metadata To Store Later

- asset id
- asset type
- source
- license
- consent note
- associated scene
- generated by engine/model
- benchmark verdict reference
- created_at

## Non-Negotiable Rules

- No celebrity voice cloning.
- No real-person voice cloning without explicit permission.
- No reference images of real people without permission.
- No generated media files in Git.
- No secret/license keys in docs.
- No browser access directly to AI Server services.
