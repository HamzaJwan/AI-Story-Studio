# SOP — تشغيل المشروع مع AI Coding Tools

## الهدف
تنظيم العمل بين:
- حمزة: صاحب القرار.
- Gemini/Antigravity: المراجع.
- Codex: المنفذ.
- Ollama: مزود الذكاء المحلي.

## سير العمل المعتمد

```text
1. حمزة يفتح المشروع في VS Code
2. يملأ .env محلياً
3. Gemini يراجع docs والخطة
4. Codex ينفذ Phase محددة
5. تشغيل تحقق محلي
6. Gemini يراجع التغيير
7. حمزة يقرر الانتقال للمرحلة التالية
```

## قبل كل مهمة

- اقرأ `CURRENT_STAGE_SUMMARY.md`.
- اقرأ `DECISION_LOG.md`.
- حدد المرحلة الحالية.
- حدد نطاق المهمة.
- لا تلمس ملفات خارج النطاق.

## الممنوع

- بناء تطبيق كامل دفعة واحدة.
- إضافة DB قبل الحاجة.
- Celery/Redis في أول MVP بدون سبب.
- ربط مباشر بنموذج واحد داخل الكود.
- مفاتيح API في frontend.
- توليد فيديو AI قبل نجاح الصوت والصور.

## قاعدة التنفيذ الصغيرة

كل مهمة يجب أن تكون:
- واضحة.
- قابلة للاختبار.
- لا تغير أكثر من 5-10 ملفات قدر الإمكان.
- تنتهي بتحديث docs.
